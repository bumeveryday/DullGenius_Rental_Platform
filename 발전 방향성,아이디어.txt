============ 💻 개발자/과제 제출용 기술적 제안 ============
[높은 점수 & 기술적 완성도를 위한 제안]

1. 데이터 시각화 (Admin Dashboard Upgrade)
   - 현재 대쉬보드가 텍스트 위주임.
   - 'Recharts'나 'Chart.js' 라이브러리를 도입하여 "일별 대여량", "인기 장르 분포" 등을 그래프로 보여주면 시각적 임팩트가 매우 큼.

2. 자동 알림 시스템 (Email Notification)
   - Google Apps Script의 강점인 `MailApp` 활용.
   - 매일 자정 트리거를 돌려 "반납 기한이 1일 남은 유저"에게 자동으로 이메일/문자를 발송하는 기능.
   - 기술적으로 간단하면서도 "실제 서비스다운" 느낌을 줌.

3. React Query (TanStack Query) 도입
   - 현재 `useEffect`로 데이터를 가져오고 있는데, 캐싱과 로딩 상태 관리가 수동적임.
   - React Query를 도입하면 데이터 캐싱, 자동 갱신(Refetch), 로딩 처리가 훨씬 세련되게 변함. (코드량 감소 + 성능 최적화 어필 가능)

4. TypeScript 마이그레이션
   - 현재 JS로 되어 있으나, TS로 변환하면 "타입 안정성"과 "대규모 프로젝트 관리 능력"을 보여줄 수 있음. (가산점 요소)

5. CI/CD 파이프라인 구축
   - GitHub Actions를 사용하여 코드를 푸시하면 자동으로 Vercel에 배포되도록 설정.
   - "자동화된 배포 환경"을 구축했다는 점에서 엔지니어링 역량 어필 가능.

6. 서버/DB 아키텍처 개선 (Supabase 마이그레이션) ★★★
   - 문제: 현재 Google Sheets 기반은 동시 접속자 10명만 넘어도 락(Lock) 걸리고 느려짐. 학교 쉬는 시간에 QR코드로 30명이 동시에 접속하면 터질 확률 99%.
   - 해결: Supabase (무료 Firebase 대안) 도입.
   - 장점:
     1. 엑셀 같은 UI 제공 (관리 쉬움)
     2. 진짜 DB라서 수천 명 동시 접속도 거뜬함
     3. 실시간 기능(누가 빌려가면 내 화면에서도 바로 '대여중' 뜸) 구현 가능
   - 비용: 학교 프로젝트 규모에서는 평생 무료. (PC 켜둘 필요 없음)
